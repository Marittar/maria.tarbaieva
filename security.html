Personas

Together with the design team, we conducted 1:1 user interviews with internal users and pentesters. In regards to customers studies, we collaborated with sales and customer success teams by jumping on calls with customers or providing a list of questions in order to capture users' data. We stored all user interview data in a single place in order to share it within the organisation.

Pentest state flow

This flow involves all user personas by conducting user interviews and moderated user testing and contextual inquiries to capture inefficient user interactions and simplify or automate them.

Special notes study

During the user testing, observing interactions of different user groups, I've noticed the following sequence of manual interactions: JT (pentest coordinator persona) fills in requirements in the following text field.

Christina (CSM persona) double checks these requirements with JT and copies the data to the comments section of our internal tool (admin application).

Then Rory (TPM persona) checks requirements and copies them from the comments to the notes in a posting to pentesters.

Nolan (Lead pentester persona) Checks the posting list and applies for a pentest based on the notes in the postings.

The discovery about frictions between user personas helped to build an idea about clear communications between customers and pentesters excluding such third parties as CSM and TPM and helping them focusing more on communication with the customers instead of repetitive actions. The solution was to handover this responcibility to a customer with a clear communication - this information is for pentesters.
